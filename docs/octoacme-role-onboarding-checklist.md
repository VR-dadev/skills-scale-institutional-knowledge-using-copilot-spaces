
# OctoAcme Role Onboarding Checklist

This checklist helps new team members understand their role, responsibilities, and key interactions within the OctoAcme project management framework.

## For All Roles

- [ ] Read `octoacme-project-management-overview.md` to understand the framework
- [ ] Review `octoacme-roles-and-personas.md` to understand all team roles
- [ ] Identify your role and read the full persona description
- [ ] Schedule a 1:1 with your role peer or lead to discuss:
  - Day-to-day responsibilities and expectations
  - Key decisions you'll own
  - Escalation paths and who to consult
  - Communication channels and cadence
- [ ] Review the project charter and current project status
- [ ] Set up calendar invites for recurring meetings (standups, syncs, demos)
- [ ] Add yourself to relevant communication channels (Slack, email lists)
- [ ] Review any ongoing risks or blockers in the Risk Register

## For Project Managers

- [ ] Review `octoacme-project-planning.md` to understand planning activities
- [ ] Review `octoacme-execution-and-tracking.md` to understand execution workflows
- [ ] Review `octoacme-risks-and-communication.md` to understand escalation and reporting
- [ ] Understand the project board setup and column definitions
- [ ] Learn the current sprint or milestone schedule
- [ ] Meet with the Product Manager to align on priorities and communication
- [ ] Schedule stakeholder kick-off meeting
- [ ] Establish communication rhythm (weekly status, sprint cadence, escalation process)

## For Product Managers

- [ ] Review `octoacme-project-initiation.md` to understand business case development
- [ ] Review `octoacme-project-planning.md` to understand backlog management
- [ ] Understand success metrics and how they're measured
- [ ] Review current roadmap and feature priorities
- [ ] Meet with the Project Manager to align on delivery timeline
- [ ] Establish user research and feedback processes
- [ ] Set up dashboard or tools for tracking success metrics

## For Developers

- [ ] Review `octoacme-execution-and-tracking.md` to understand pull request and quality processes
- [ ] Understand the branching strategy and PR requirements
- [ ] Set up your development environment following repo documentation
- [ ] Review the Definition of Done for this project
- [ ] Understand how to run tests and linting locally
- [ ] Participate in sprint planning to understand estimation process
- [ ] Meet with tech lead to discuss architecture and design patterns

## For QA/Testing Leads

- [ ] Review `octoacme-execution-and-tracking.md` to understand quality standards
- [ ] Review `octoacme-project-planning.md` to understand test planning
- [ ] Understand the Definition of Done and acceptance criteria requirements
- [ ] Review existing test plans and test automation infrastructure
- [ ] Meet with Product Manager to understand feature priorities and edge cases
- [ ] Coordinate with DevOps on staging environment setup
- [ ] Establish defect triage and reporting process

## For UX Designers

- [ ] Review `octoacme-project-planning.md` to understand feature planning
- [ ] Review `octoacme-execution-and-tracking.md` to understand delivery workflow
- [ ] Meet with Product Manager to understand product vision and user needs
- [ ] Set up design collaboration tools and share access with team
- [ ] Understand the design system and existing UI patterns
- [ ] Establish design review cadence with engineering and product
- [ ] Learn how user feedback is collected and incorporated

## For Business Analysts

- [ ] Review `octoacme-project-initiation.md` to understand business case
- [ ] Review `octoacme-project-planning.md` to understand requirements gathering
- [ ] Meet with Product Manager to understand business priorities
- [ ] Understand stakeholder landscape and communication needs
- [ ] Review current requirements and backlog
- [ ] Establish requirements refinement and validation process
- [ ] Coordinate with QA on test case development

## For DevOps Engineers

- [ ] Review `octoacme-release-and-deployment.md` to understand deployment process
- [ ] Review `octoacme-execution-and-tracking.md` to understand CI/CD requirements
- [ ] Meet with tech lead to understand infrastructure and deployment targets
- [ ] Set up CI/CD pipelines and automation tooling
- [ ] Document deployment runbook and rollback procedures
- [ ] Establish monitoring and alerting for key systems
- [ ] Coordinate with Security Lead on infrastructure security controls

## For Security Leads

- [ ] Review `octoacme-risks-and-communication.md` to understand risk management
- [ ] Review `octoacme-release-and-deployment.md` to understand security gates
- [ ] Meet with Product Manager and tech lead to understand security requirements
- [ ] Conduct threat modeling for the project
- [ ] Establish secure coding guidelines and code review process
- [ ] Coordinate with DevOps on security scanning and compliance controls
- [ ] Set up vulnerability disclosure and incident response procedures

## Project Kick-Off Meeting (All Roles)

Use this agenda to kick off the project with all team members:

- [ ] Distribute project charter and success metrics
- [ ] Review project goals and timeline
- [ ] Introduce all team members and discuss role expectations
- [ ] Discuss key dependencies and communication cadence
- [ ] Review initial risk register and mitigation plans
- [ ] Answer questions and address concerns
- [ ] Schedule next sync and confirm availability

---

**Note:** Customize this checklist per project and team structure. Remove items that don't apply to your context.
